# FinanceBotLite 리부트 실행 계획

기준일: 2026-04-24 (UTC)

## 현재 상태 업데이트
- 탭형 앱 구조, 기본 미니미 커스터마이징, 샵/코인 루프, 파스텔 CSS 테마는 구현 완료.
- 외부 API 연동은 fallback 기반으로 안전하게 동작하도록 반영 완료.
- 다음 단계는 **문서 명세 고도화 + 실제 결제 연동 + 지표 수집**이다.

## 1. 리부트 목표
- 기존의 단일 질문 기반 추천 앱을, **재방문성(습관)**과 **결제 전환**이 가능한 금융 라이프스타일 앱으로 전환한다.
- 핵심 방향:
  1. 실사용성: 추천 신뢰도, 비교 가능성, 목표 추적
  2. Lovable 경험: 아바타/미니룸 커스터마이징, 감성 카피, 보상 루프
  3. 수익화: 인앱 결제(구독 + 단건 테마팩)

---

## 2. 제품 콘셉트
- 한 줄 정의: **"금융 습관을 키우면 내 미니미와 방도 성장하는 앱"**
- 주요 사용자:
  - 사회초년생(저축/비상금)
  - 직장인 재테크 입문자(리스크 낮은 분산/적립)

---

## 3. MVP 범위 (6주)

### 포함
- 온보딩형 추천 플로우(목적/기간/리스크/납입액)
- 추천 카드 + 비교표 + 주의사항
- 아바타 기본 커스터마이징(헤어/상의/배경)
- 미션 기반 코인 보상
- 샵(무료/유료 아이템) + Paywall
- 핵심 이벤트 트래킹

### 제외 (MVP 이후)
- 실시간 멀티 방문자 미니룸
- 복잡한 가구 배치 에디터
- 고급 커뮤니티 피드

---

## 4. 기능 트랙

### A. 추천 신뢰 트랙
1. 입력 스키마 정규화
2. 추천 사유 3가지 + 주의사항 2가지 노출
3. 상위 3개 비교표

### B. Lovable 트랙
1. 아바타 생성/장착
2. 미니룸 배경 변경
3. 체크인/미션/배지

### C. 결제 트랙
1. Free/Pro 구분
2. 유료 테마팩 단건 구매
3. 결제 성공 후 권한 잠금해제

### D. 데이터 트랙
1. 이벤트: onboarding_complete, recommendation_view, mission_complete, paywall_view, purchase_success
2. 주간 KPI 대시보드

---

## 5. 주차별 실행

## Week 1 — 기획 확정
- PRD/요구사항/와이어 초안
- 결제 정책 초안
- 이벤트 스키마 확정

## Week 2 — 추천 UX 개편
- 탭 구조 도입(홈/미니룸/샵/프로필)
- 온보딩 플로우 구현
- 추천 카드/비교표 구현

## Week 3 — 커스터마이징 1차
- 아바타 상태 모델
- 기본 아이템 15개
- 장착/미리보기 UI

## Week 4 — 샵/결제 1차
- 샵 UI
- Free/Pro 권한 분기
- 결제 성공 처리 훅

## Week 5 — 보상 루프
- 미션/코인/배지
- 주간 인사이트 카드

## Week 6 — 안정화/베타
- 버그 수정
- 카피/Paywall A/B 1차
- 베타 테스트 5~10명

---

## 6. 품질 기준 (Definition of Done)
- 주요 플로우(온보딩→추천→저장→샵→구매)가 끊기지 않는다.
- 결제 후 유료 권한이 즉시 반영된다.
- 핵심 이벤트 누락률 5% 미만.
- UI는 모바일 기준 1초대 초기 렌더를 목표로 한다.

---

## 7. 리스크와 대응
- 결제 정책/스토어 심사 지연: 웹 결제 우선 PoC 병행
- 게임성 과다로 금융 신뢰 저하: 금융 정보 영역의 가독성/고지 고정
- 아이템 제작 비용 증가: 시즌당 아이템 수 제한(20~30)
